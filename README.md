## Overview

A C console application that simulates a simple **text editor**. Words from a file are stored in a **doubly linked list**. Every insert and delete is recorded in an **Undo stack** so it can be reversed; undone operations are pushed to a **Redo stack**. A **queue** is used to buffer multiple words entered at once before inserting them into the list.

## Menu Options

| Option | Description |
|---|---|
| 1 | Load `input.txt` into the doubly linked list |
| 2 | Print all words in the current text |
| 3 | Insert word(s) ŌĆö choose: `between`, `first`, `last`, or `edit` |
| 4 | Remove a word ŌĆö choose: `deleteString` (full remove) or `deleteword` (replace with shorter) |
| 5 | Undo last operation |
| 6 | Redo last undone operation |
| 7 | Print the Undo and Redo stacks |
| 8 | Save current text to `output.txt` |
| 9 | Exit |

### Insert modes (option 3)
| Mode | Behaviour |
|---|---|
| `between` | Insert after a specified existing word |
| `first` | Insert at the beginning of the text |
| `last` | Append to the end of the text |
| `edit` | Replace an existing word with one of equal or greater length |

### Delete modes (option 4)
| Mode | Behaviour |
|---|---|
| `deleteString` | Remove the word entirely |
| `deleteword` | Replace the word with a shorter version |

## Data Structures

### Doubly Linked List (text storage)
Each word occupies one node. A **sentinel head node** (no data) simplifies boundary cases.
```c
typedef struct node {
    char Data[1000];
    struct node* Next;
    struct node* pre;
} node;
```

### Undo Stack
Stores every change so it can be reversed. Fields: original word, edited word, type of change, position, character index.
```c
typedef struct Undo {
    char undo[400];          // original (before change)
    char edit[400];          // new value (after change)
    char TypeOfChange[400];  // "Insert", "Remove", "Insert(edit)", "Insert(delete)"
    int  position;
    int  index;
    struct Undo* next;
} Undo;
```

### Redo Stack
Same structure; populated when Undo is performed.

### Queue
Buffers multiple words entered in one go before inserting them one by one into the list.
```c
typedef struct Queue {
    char newString[50];
    struct Queue* next;
} Queue;
```

## Change Types (Undo/Redo)

| TypeOfChange | Meaning | Undo action | Redo action |
|---|---|---|---|
| `Insert` | Word was added | Delete it | Re-insert it |
| `Remove` | Word was deleted | Re-insert it | Delete it again |
| `Insert(edit)` | Word replaced with longer | Delete new, re-insert original | Delete original, re-insert new |
| `Insert(delete)` | Word replaced with shorter | Delete new, re-insert original | Delete original, re-insert new |

## Bug Fixes Applied

| # | Location | Bug | Fix |
|---|---|---|---|
| 1 | `index()` function | Name conflicts with POSIX `index()` from `<strings.h>` ŌĆö **compile error** | Renamed to `getIndex()` everywhere |
| 2 | `saveToFile` | Used before declaration ŌĆö **implicit declaration warning** | Added forward declaration at top |
| 3 | `InsertLast` | Dereferenced `head` when it was `NULL` (crash on empty list) | Added null check; creates sentinel node if missing |
| 4 | `PerformRedo` ŌĆö `Insert(edit)` case | Used `top->edit` (from Undo stack!) instead of `top1->edit` (Redo stack) ŌĆö wrong word re-inserted | Changed to `top1->edit` |
| 5 | `PerformRedo` ŌĆö `Insert(delete)` case | Checked `top->TypeOfChange` (Undo stack) instead of `top1->TypeOfChange` ŌĆö condition never matched | Changed to `top1->TypeOfChange` |
| 6 | `EditWord` | Called `FindNode(head, ...)` starting from sentinel ŌĆö would never match data node | Changed to `FindNode(head->Next, ...)` |
| 7 | `DeleteChar` | Same sentinel search issue as above | Changed to `FindNode(head->Next, ...)` |
| 8 | `readFile` | Used `fscanf(in, "%s\n", line)` ŌĆö skips whitespace differently per platform; misses multi-word lines | Changed to `fgets` for reliable full-line reading |
| 9 | `DeleteChar` | Called `Enqueue` twice for the same word ŌĆö inserted replacement twice into queue | Removed duplicate `Enqueue` call |
| 10 | `InsertNewWord` ŌĆö `between` mode | Did not drain queue when target word was not found, leaving stale entries | Added queue drain on not-found path |
| 11 | All `getIndex` call sites | Called the old `index()` name (broken after rename) | Updated all call sites to `getIndex()` |
